## Overview

nene.js provides a unified context system that seamlessly shares data between your frontend and backend. This eliminates the need for manual state synchronization and reduces boilerplate code.

<Callout type="info">
  Unified Context is one of the key features that makes nene.js unique - write
  once, use everywhere.
</Callout>

## Request Context

Access request information anywhere in your application using the unified context:

```typescript
import { useContext } from "nene";

// In a controller
@Controller("/api/posts")
export class PostController {
  @Get("/")
  async getPosts() {
    const { user, session } = useContext();

    if (!user) {
      throw new UnauthorizedException();
    }

    return await this.postService.findByUser(user.id);
  }
}
```

## Context in Frontend

The same context is available in your React components:

```tsx
"use client";

import { useNeneContext } from "nene/react";

export function UserProfile() {
  const { user, isLoading } = useNeneContext();

  if (isLoading) return <Skeleton />;
  if (!user) return <LoginPrompt />;

  return <div>Welcome, {user.name}!</div>;
}
```

## Available Context

| Property  | Type                | Description                |
| --------- | ------------------- | -------------------------- |
| `user`    | `User \| null`      | Current authenticated user |
| `session` | `Session \| null`   | Current session data       |
| `locale`  | `string`            | Current locale             |
| `theme`   | `'light' \| 'dark'` | Current theme              |

## Custom Context

You can extend the context with your own data:

```typescript
// nene.config.ts
import { defineConfig } from "nene";

export default defineConfig({
  context: {
    extend: async (ctx) => ({
      ...ctx,
      tenant: await getTenantFromRequest(ctx.request),
    }),
  },
});
```

Then access it anywhere:

```typescript
const { tenant } = useContext();
```
