## 개요

일반적인 nene.js 프로젝트는 관심사를 분리하면서도 통합된 개발 경험을 유지하는 구조화된 레이아웃을 따릅니다:

```text
my-nene-app/
├── src/
│   ├── app/              # Next.js App Router 페이지
│   ├── controllers/      # nene.js 컨트롤러
│   ├── services/         # 비즈니스 로직 서비스
│   ├── hooks/            # 자동 생성된 훅
│   └── components/       # React 컴포넌트
├── nene.config.ts        # nene.js 설정
├── next.config.js        # Next.js 설정
└── package.json
```

## 주요 디렉토리

### `src/controllers/`

nene.js 컨트롤러가 포함됩니다. API 엔드포인트를 정의하고 해당하는 프론트엔드 훅을 생성합니다.

```typescript
// src/controllers/post.controller.ts
@Controller("/api/posts")
export class PostController {
  @Get("/")
  async getPosts() {
    /* ... */
  }

  @Post("/")
  async createPost(@Body() data: CreatePostDto) {
    /* ... */
  }
}
```

<Callout type="info">
  이 디렉토리에 배치된 컨트롤러는 자동으로 검색되고 등록됩니다.
</Callout>

### `src/hooks/`

각 컨트롤러 메서드에 대한 타입 안전한 React 훅이 포함된 자동 생성 디렉토리입니다.

<Callout type="warning">
  이 디렉토리의 파일을 수동으로 편집하지 마세요 - 훅이 재생성되면
  덮어씌워집니다.
</Callout>

### `src/services/`

컨트롤러 간에 공유할 수 있는 비즈니스 로직입니다. 의존성 주입을 사용합니다:

```typescript
// src/services/user.service.ts
import { Injectable } from "nene/decorators";

@Injectable()
export class UserService {
  async findById(id: string) {
    // 비즈니스 로직을 여기에 작성
  }
}
```

### `src/app/`

React 페이지와 레이아웃을 위한 표준 Next.js App Router 디렉토리입니다.

## 설정 파일

| 파일             | 설명                    |
| ---------------- | ----------------------- |
| `nene.config.ts` | nene.js 프레임워크 설정 |
| `next.config.js` | Next.js 설정            |
| `tsconfig.json`  | TypeScript 설정         |

## 모범 사례

1. **기능별 그룹화** - 더 큰 앱의 경우 기능별로 컨트롤러와 서비스를 그룹화하는 것을 고려하세요
2. **컨트롤러는 가볍게** - 비즈니스 로직을 서비스로 이동하세요
3. **DTO 사용** - 유효성 검사를 위해 명확한 데이터 전송 객체를 정의하세요
