## 개요

nene.js는 프론트엔드와 백엔드 간에 데이터를 원활하게 공유하는 통합 컨텍스트 시스템을 제공합니다. 이는 수동 상태 동기화의 필요성을 없애고 보일러플레이트 코드를 줄입니다.

<Callout type="info">
  통합 컨텍스트는 nene.js를 독특하게 만드는 핵심 기능 중 하나입니다 - 한 번
  작성하고 어디서나 사용하세요.
</Callout>

## 요청 컨텍스트

통합 컨텍스트를 사용하여 애플리케이션 어디서나 요청 정보에 접근하세요:

```typescript
import { useContext } from "@nene/core";

// 컨트롤러에서
@Controller("/api/posts")
export class PostController {
  @Get("/")
  async getPosts() {
    const { user, session } = useContext();

    if (!user) {
      throw new UnauthorizedException();
    }

    return await this.postService.findByUser(user.id);
  }
}
```

## 프론트엔드의 컨텍스트

React 컴포넌트에서도 동일한 컨텍스트를 사용할 수 있습니다:

```tsx
"use client";

import { useNeneContext } from "@nene/react";

export function UserProfile() {
  const { user, isLoading } = useNeneContext();

  if (isLoading) return <Skeleton />;
  if (!user) return <LoginPrompt />;

  return <div>환영합니다, {user.name}님!</div>;
}
```

## 사용 가능한 컨텍스트

| 속성      | 타입                | 설명               |
| --------- | ------------------- | ------------------ |
| `user`    | `User \| null`      | 현재 인증된 사용자 |
| `session` | `Session \| null`   | 현재 세션 데이터   |
| `locale`  | `string`            | 현재 로케일        |
| `theme`   | `'light' \| 'dark'` | 현재 테마          |

## 커스텀 컨텍스트

자신만의 데이터로 컨텍스트를 확장할 수 있습니다:

```typescript
// nene.config.ts
import { defineConfig } from "@nene/core";

export default defineConfig({
  context: {
    extend: async (ctx) => ({
      ...ctx,
      tenant: await getTenantFromRequest(ctx.request),
    }),
  },
});
```

그런 다음 어디서나 접근하세요:

```typescript
const { tenant } = useContext();
```
