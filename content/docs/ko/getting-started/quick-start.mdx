## 개요

이 가이드에서 배우게 될 내용:

- 타입 안전한 엔드포인트가 있는 컨트롤러 생성
- 프론트엔드에서 자동 생성된 훅 사용
- 개발 서버 실행

<Callout type="info">
  이 가이드는 nene.js가 이미 설치되어 있다고 가정합니다. 아직 설치하지 않았다면
  [설치](/docs/ko/getting-started/installation) 가이드를 먼저 확인하세요.
</Callout>

## 단계별 가이드

<Steps>
  <Step title="컨트롤러 생성">
    백엔드 로직을 처리하고 프론트엔드 훅을 자동으로 생성하는 새 컨트롤러를 만듭니다:

    ```typescript
    // src/controllers/user.controller.ts
    import { Controller, Get, Query } from '@nene/core';

    @Controller('/api/users')
    export class UserController {
      @Get('/')
      async getUsers(@Query('search') search?: string) {
        // 백엔드 로직을 여기에 작성
        return { users: [] };
      }
    }
    ```

  </Step>

  <Step title="자동 생성된 훅 사용">
    nene.js는 프론트엔드를 위한 타입 안전한 훅을 자동으로 생성합니다:

    ```tsx
    // src/app/page.tsx
    'use client';

    import { useUsers } from '@/hooks/user';

    export default function UsersPage() {
      const { data, isLoading } = useUsers({ search: 'john' });

      if (isLoading) return <div>로딩 중...</div>;

      return (
        <ul>
          {data?.users.map(user => (
            <li key={user.id}>{user.name}</li>
          ))}
        </ul>
      );
    }
    ```

  </Step>

  <Step title="개발 서버 실행">
    개발 서버를 시작하여 변경 사항을 실시간으로 확인하세요:

    ```bash
    npm run dev
    ```

    [http://localhost:3000](http://localhost:3000)을 열어 애플리케이션을 확인하세요.

  </Step>
</Steps>

<Callout type="tip" title="핫 리로드">
  개발 서버는 컨트롤러를 수정하면 자동으로 훅을 재생성합니다. 수동 작업이 필요
  없습니다!
</Callout>
